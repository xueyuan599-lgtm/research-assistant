# 团队协作部署指南

## 如何让队友安全地使用 Research Assistant 项目

---

## 一、安全风险全景图

在共享项目前，先搞清楚哪些是**必须公开的**、哪些是**绝对不能公开的**：

### 🟢 安全可共享（项目核心价值）

| 内容 | 路径 | 说明 |
|------|------|------|
| Agent 定义文件 | `agents/*.md` | 43个Agent的职责、输入输出定义 |
| 工作流规则 | `.claude/rules/*.md` | 7条核心规则 |
| 知识库 | `knowledge/**/*.md` | 60+算法文档、Kaggle模式库 |
| 工作流 | `workflows/*.md` | 动态管线协议 |
| 代码输出 | `outputs/**/*.py` | 所有实战案例代码 |
| 项目说明 | `CLAUDE.md` | 项目描述 |

### 🔴 禁止共享（含个人隐私/凭证）

| 内容 | 位置 | 风险 |
|------|------|------|
| **Kaggle API Token** | 父项目的 `settings.local.json` | ⚠️ 他人可用你的Kaggle账号 |
| **个人路径** | `settings.local.json` 含 `C:\Users\lenovo\` | 泄露用户名和目录结构 |
| **MATLAB 路径** | `settings.local.json` 含 MATLAB 工具链配置 | 本地环境信息 |
| **Claude Code 权限** | `settings.local.json` 的 allow 列表 | 暴露你的操作习惯 |

## 二、一键清理敏感信息

### 步骤1：把 settings.local.json 移出 git 追踪

```bash
# 进入项目目录
cd E:\wuyi\数学建模半自动\research-assistant

# 从 git 中移除（但保留本地文件）
git rm --cached .claude/settings.local.json

# 添加到 .gitignore
echo ".claude/settings.local.json" >> .gitignore
echo ".claude/settings.json" >> .gitignore
echo "catboost_info/" >> .gitignore

# 提交更改
git commit -m "chore: remove personal settings from git tracking"

# 推送到远程仓库
git push
```

### 步骤2：创建共享模板配置文件

创建 `.claude/settings.template.json`（见下文**模板内容**），告诉 Claude Code 项目需要哪些权限。

### 步骤3：检查是否有其他隐私泄露

```bash
# 搜索是否有硬编码的密钥
grep -rn "api_key\|API_KEY\|token\|password\|secret" \
  --include="*.py" --include="*.md" --include="*.json" \
  agents/ knowledge/ workflows/ outputs/
```

## 三、共享模板配置

创建以下文件作为团队共享的参考配置：

**`.claude/settings.template.json`：**

```json
{
  "permissions": {
    "allow": [
      "Bash(python *)",
      "Bash(pip install *)",
      "Bash(git *)",
      "Bash(curl *)",
      "WebFetch(domain:openalex.org)",
      "WebFetch(domain:export.arxiv.org)",
      "WebFetch(domain:github.com)",
      "WebFetch(domain:raw.githubusercontent.com)",
      "Bash(Rscript *)"
    ]
  },
  "env": {}
}
```

> **说明**：队友拿到后复制为 `settings.local.json` 并根据自己的环境调整（如 Python 路径、是否装 R/MATLAB 等）。

## 四、团队部署方案

### 方案 A：GitHub 仓库共享（推荐 ⭐）

这是最简单、最规范的方案。

```
你（负责人）                               队友
    │                                        │
    ├─ git rm settings.local.json             │
    ├─ 推送到 GitHub                          │
    │                                        ├─ git clone <仓库URL>
    │                                        ├─ 安装 Claude Code
    │                                        ├─ cp settings.template.json .claude/settings.local.json
    │                                        └─ claude → 直接使用
    │
    └─ 队友无需任何 API 密钥
        （项目使用的 OpenAlex/ArXiv 都是免费的）
```

**具体操作：**

```bash
# 你这边（已清理完敏感信息后）
git remote add origin https://github.com/<你的用户名>/research-assistant.git
git push -u origin main

# 队友那边
git clone https://github.com/<你的用户名>/research-assistant.git
cd research-assistant

# 安装 Claude Code（队友用自己的账号）
npm install -g @anthropic-ai/claude-code

# 复制配置文件
cp .claude/settings.template.json .claude/settings.local.json

# 安装 Python 依赖
pip install numpy scipy pandas scikit-learn statsmodels matplotlib seaborn xgboost lightgbm

# 启动
claude
```

### 方案 B：压缩包分享（不用 GitHub）

```bash
# 你这边：创建一个干净的分享包
cd E:\wuyi\数学建模半自动
git clone research-assistant research-assistant-clean  # 克隆一份
cd research-assistant-clean

# 移除个人配置
git rm --cached .claude/settings.local.json
echo ".claude/settings.local.json" >> .gitignore
git commit -m "remove personal settings"

# 创建模板配置（见上）
code .claude/settings.template.json

# 打包（排除 git 历史以减小体积）
cd ..
tar -czf research-assistant-share.tar.gz \
  --exclude=".git" \
  --exclude="__pycache__" \
  --exclude="catboost_info" \
  research-assistant-clean/

# 把这个压缩包发给队友即可
```

## 六、FAQ

### Q1: 队友能看到我的 Kaggle API Token 吗？
**A:** 不会。Kaggle Token 在父项目的 `settings.local.json` 里，那个文件**不在** `research-assistant` 这个 git 仓库里。只要你不手动复制那个文件，队友绝对看不到。

### Q2: 队友没有 MATLAB 能用吗？
**A:** 能用。MATLAB 只是可选工具。核心功能（文献检索、数据分析、可视化、论文排版）全部基于 Python，不依赖 MATLAB。

### Q3: 队友需要自己的 Claude Code 账号吗？
**A:** 是的。每个使用者需要自己的 Claude Code 订阅（队友自己安装）。项目文件本身是免费的。

### Q4: 队友修改了代码怎么同步回来？
**A:** 最安全的做法：队友 fork 你的仓库 → 修改 → 提 Pull Request → 你审查后合并。这样既能协作，又不会搞乱主仓库。

### Q5: 如果队友不小心提交了个人信息到仓库怎么办？
**A:** 在 `.gitignore` 中加：
```
# 个人信息保护
settings.local.json
*.local.json
.env
*.env
```
