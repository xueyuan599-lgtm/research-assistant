---

## 一、项目概述

本项目"科研全流程智能辅助助手——Research Assistant"面向科学研究全过程的效率瓶颈，基于大语言模型驱动的多智能体（Multi-Agent）架构，构建覆盖科研知识问答、文献智能检索、论文选题分析、数据处理与可视化、实验流程优化、论文格式排版、知识沉淀等七大功能模块的智能辅助系统。

技术架构上，系统设计了智能协调器（Orchestrator）统一调度各领域智能体，支持动态管线组装、自动上下文管理、质量门禁验证与对抗式审查。

## 三、关键技术实现细节

### 3.1 多智能体架构

```
用户输入
  │
  ├─→ 秘书 Agent (secretary.md)
  │     任务分解方案 + 用户确认
  │
  ├─→ Orchestrator 调度
  │     意图识别 → 路由 → 管线组装 → 上下文监控
  │
  ├─→ 各领域 Agent 并行/串行执行
  │     Literature / Topic / DataViz / Experiment / Format / QA / Algorithm
  │
  ├─→ 共享记忆传递 (memory_chain)
  │     发现 → 数据路径 → 警告 → 交接备注
  │
  ├─→ Critic 对抗式审查
  │     完整性检查 → 合理性检查 → 有问题→Fixer循环
  │
  └─→ 交付
      代码 + 结果 + 报告
```

### 3.2 上下文管理机制

当任务执行超过上下文容量的50%时，系统自动触发：
1. 写检查点（`checkpoint.write_checkpoint()`）
2. 创建新Agent从断点继续
3. 关键上下文以摘要形式传递

### 3.3 质量门禁体系

| 门禁层级 | 检查内容 | 通过标准 |
|---------|---------|---------|
| 完整性检查 | 所有输出文件存在且非空 | 无缺失 |
| 合理性检查 | 数值无异常、图表可渲染、文本自洽 | 无异常 |
| 对抗式审查 | Critic-Fixer 交替检查 | 最多3轮 |
| 写作质量标准 | 禁用词、句长、句式、信息密度 | ≥6/10 |
