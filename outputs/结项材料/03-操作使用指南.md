# 操作使用指南

## 科研全流程智能辅助助手 —— Research Assistant

---

## 一、环境准备

### 1.1 安装 Claude Code

```bash
# 通过 npm 安装
npm install -g @anthropic-ai/claude-code
```

### 1.2 克隆项目代码

```bash
git clone <项目仓库地址>
cd research-assistant
```

### 1.3 安装 Python 依赖

```bash
# 建议使用 conda 创建独立环境
conda create -n research-assistant python=3.12
conda activate research-assistant

# 安装核心依赖
pip install numpy scipy pandas scikit-learn statsmodels
pip install matplotlib seaborn plotnine
pip install xgboost lightgbm catboost
pip install python-docx openpyxl
pip install sympy requests tqdm

# 可选安装（特定领域）
# R 集成：rpy2
# MATLAB 集成：matlab.engine
```

## 二、启动系统

### 2.1 项目初始化

首次使用时，在项目根目录打开终端，Claude Code 会自动加载项目配置：

```bash
cd research-assistant
claude
```

### 2.2 核心命令

| 命令 | 功能 | 示例 |
|------|------|------|
| `/research <需求>` | 启动科研辅助流水线 | `/research 帮我做一个DID分析的文献综述` |
| 直接输入需求 | 自动触发秘书Agent分解任务 | `用这份数据做回归分析并画图` |
| 自然语言交互 | 与系统对话式交互 | `解释一下双重机器学习的原理` |

> **注意**：系统采用多Agent架构，输入任务后会启动秘书Agent进行任务分解。你将看到任务分解方案，确认后系统自动调度执行。

## 三、典型使用场景

### 场景1：文献自动综述

**目标：** 输入研究主题，自动生成结构化文献综述。

**操作步骤：**

```
第1步：输入需求
  └─ "/research 帮我检索2024年关于双重机器学习(DML)方法的前沿文献，生成综述"

第2步：秘书Agent输出分解方案（示例）
  ├─ Literature Search Agent: 搜索DML相关文献（可并行）
  ├─ Literature Screening Agent: 筛选高质量文献
  └─ Literature Synthesis Agent: 生成综述报告

第3步：用户确认
  └─ 确认分解方案 → 系统自动执行

第4步：系统输出
  └─ 结构化综述报告（含各方法对比、关键发现、未来方向）
```

**预期输出：** 结构化综述文档，包含研究主题分布、方法论分类、关键文献列表。

### 场景2：数据分析与可视化

**目标：** 输入原始数据和分析需求，自动完成清洗→建模→可视化全流程。

**操作步骤：**

```
第1步：输入需求
  └─ "使用这份面板数据（data.csv），做双向固定效应回归分析，
       生成系数估计表、平行趋势检验图、安慰剂检验图"

第2步：秘书Agent分解
  ├─ Data Cleaning Agent: 数据清洗与格式检查
  ├─ Modeling Agent: 双向固定效应估计
  ├─ Visualization Agent: 生成出版级图表
  └─ Interpretation Agent: 结果解读

第3步：系统自动执行 → 输出
  ├─ 回归系数表（表格 + CSV）
  ├─ 平行趋势检验图（PNG, 300dpi）
  ├─ 安慰剂检验图（PNG, 300dpi）
  └─ 结果解读文本
```

**预期输出：** 统计分析结果 + 出版级图表 + 数据解读。

### 场景3：实验方案设计

**目标：** 自动设计实验方案并进行参数优化。

**操作步骤：**

```
第1步：输入需求
  └─ "我用RDD方法研究奖学金对成绩的影响，帮我做带宽选择的敏感性分析"

第2步：系统输出
  ├─ RDD模型估计结果
  ├─ 不同带宽下的估计系数对比图
  ├─ 最优带宽选择建议
  └─ 敏感性分析报告
```

### 场景4：论文格式排版

**目标：** 将论文自动适配目标期刊格式。

**操作步骤：**

```
第1步：输入需求
  └─ "将这篇论文格式化投稿《统计研究》的格式"

第2步：系统输出
  ├─ 格式化后的文档
  ├─ 参考文献格式化
  ├─ 格式问题报告
  └─ 合规检查结果
```

## 四、高级功能

### 4.1 算法创造流水线

将研究想法转化为可运行的算法：

```
输入：一个自然语言的研究方法想法
  └─ Formalizer Agent: 想法→数学问题定义
  └─ Designer Agent: 数学定义→伪代码+性质
  └─ Coder Agent: 伪代码→可运行代码+测试
  └─ Benchmark Agent: 与既有方法数值对比
  └─ Validator Agent: 实际运行验证
输出：完整算法包 + 基准对比报告 + 知识库条目
```

### 4.2 Kaggle 竞赛自动化

端到端的数据竞赛自动化流水线：

```
输入：Kaggle竞赛URL
  └─ Data Explorer Agent: 数据探查+泄漏检测
  └─ Baseline Agent: 多模型基线对比
  └─ Feature Engineer Agent: 特征工程
  └─ Model Builder Agent: 精模+调参
  └─ Ensemble Agent: 集成（Stacking+Blending）
  └─ Submission Agent: 提交+LB跟踪
输出：submission.csv + 实验日志 + 赛后复盘
```

### 4.3 共享记忆机制

多Agent协作时自动传递关键信息，避免重复劳动：

```
Agent A（数据探查）→ 发现"season列标错了"→ 写入共享记忆
Agent B（数据清洗）→ 启动时读取共享记忆 → 自动修复season列
Agent C（建模）→ 读取清洗后的数据和前置警告 → 选择合适模型
```

## 六、常见问题（FAQ）

### Q1: 系统处理长任务时会不会中断？
**A:** 系统内置智能上下文管理机制，当上下文接近容量上限时自动创建检查点并分段推进，保障长任务稳定执行。

### Q2: 我的数据会上传到外部服务器吗？
**A:** 不会。所有数据在本地处理，不上传至任何外部服务器。API调用仅用于学术文献检索等公开信息获取。

### Q3: 是否可以自定义修改Agent？
**A:** 可以。每个Agent定义为独立的`.md`文件，修改职责、输入输出、工具约束即可自定义行为。

### Q4: 系统支持中文吗？
**A:** 完全支持中文输入和输出。报告、图表标签、交互界面均为中文。

### Q5: 需要联网吗？
**A:** 核心分析功能在本地运行，但文献检索和API调用需要联网。
