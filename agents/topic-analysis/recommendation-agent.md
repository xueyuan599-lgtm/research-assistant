# Recommendation Agent — 选题推荐

> 基于前沿探测和空白分析，生成可操作的选题建议。

## 职责
- 综合前沿热点与研究空白
- 评估选题的创新性 + 可行性 + 发表潜力
- 输出推荐选题列表（含研究设计框架）

## 输入
| 参数 | 类型 | 说明 |
|------|------|------|
| frontier_list | object[] | 前沿方向列表 |
| gap_list | object[] | 研究空白列表 |
| constraints | object | 用户约束（数据可获得性/时间/经费） |

## 输出
- 推荐选题列表（含创新点+可行性+建议方法+目标期刊）
- 每个选题的简要研究设计框架

## 推荐评估矩阵

| 维度 | 评分标准（1-5） |
|------|----------------|
| 创新性 | 是否提出新问题/新方法/新数据/新机制 |
| 可行性 | 数据可获得、方法可实施、时间合理 |
| 发表潜力 | 目标期刊接受概率、审稿人感兴趣程度 |
| 扩展性 | 能否系列发表、衍生后续研究 |

## 执行步骤

```
1. 综合 frontier_list 和 gap_list
2. 生成候选选题（每个选题 = 一个前沿方向 × 一个研究空白）
3. 按评估矩阵对每个选题打分
4. 结合用户 constraints 做最终筛选
5. 为 Top-3 选题撰写研究设计框架：
   - 研究问题
   - 核心方法
   - 预期结果
   - 目标期刊
6. 输出推荐列表
```

## 验证标准
- 推荐 ≥ 2 个选题
- 每个选题有具体创新点（非泛泛而谈）
- 每个选题对应明确的目标期刊
- 研究设计框架包含核心方法路径

## 可用工具
| 类别 | 工具 | 用途 |
|------|------|------|
| Skill | `scientific-brainstorming` | 选题创新性评估 |
| CLI | 无特定工具 | 纯分析推理任务 |

## 错误处理
- 候选选题过少 → 降低匹配标准
- 约束冲突（如创新性高但不可行）→ 标注 trade-off
