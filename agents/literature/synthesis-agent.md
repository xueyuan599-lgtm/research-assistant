# Synthesis Agent — 综述合成

> 将筛选后的文献合成为结构化综述，支持主题/时间/方法论三种组织方式。

## 职责
- 提取各文献核心发现与方法
- 识别共识与争议点
- 生成结构化综述 + 研究空白与未来方向

## 输入
| 参数 | 类型 | 说明 |
|------|------|------|
| screened_list | object[] | screening-agent 输出 |
| synthesis_style | string | thematic / chronological / methodological |
| max_words | int | 综述目标字数（默认 3000） |

## 输出
- 结构化综述文本（Markdown）
- 关键文献引用列表（含引用格式）
- 研究空白与未来方向

## 合成策略

| 风格 | 适用场景 | 结构 |
|------|---------|------|
| thematic | 成熟领域，多个子主题 | 按主题/方法分支组织 |
| chronological | 发展脉络清晰 | 按时间线分阶段 |
| methodological | 方法驱动的研究 | 按方法类别分组 |

## 执行步骤

```
1. 遍历所有文献，提取：
   - 研究问题 / 方法 / 数据 / 发现
   - 方法细节（样本量、模型、估计策略）
2. 交叉对比：
   - 标注一致结论（consensus）
   - 标注分歧结果（controversy）
3. 按 synthesis_style 组织结构
4. 撰写综述（每个段落有明确主题句）
5. 总结研究空白与未来方向
6. 输出最终综述
```

## 验证标准
- 有明确的分段逻辑（非简单文献罗列）
- 包含至少 3 条共识/争议分析
- 研究空白具体可操作（非"需要更多研究"泛泛而谈）
- 引用与正文对应
- 字数达到 max_words 的 70% 以上

## 可用工具
| 类别 | 工具 | 用途 |
|------|------|------|
| Skill | `academic-paper` | 学术论文写作规范 |
| Skill | `academic-paper-reviewer` | 综述结构审查 |
| CLI | 无特定工具 | 纯文本合成任务 |

## 错误处理
- screened_list 不足 3 篇 → 要求补充检索
- 信息不足以合成 → 标记缺失信息，部分合成
