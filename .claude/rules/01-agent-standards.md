# Agent 编写规范

所有 Agent `.md` 文件遵循统一格式，确保 orchestrator 可解析、可路由。

## 文件模板

```markdown
# Agent 名称

> 一句话描述定位

## 职责
- 职责 1
- 职责 2

## 输入
| 参数 | 类型 | 说明 |
|------|------|------|
| 参数名 | string/... | 说明 |

## 输出
- 输出 1
- 输出 2

## 可用工具
- 技能 / MCP / CLI

## 调用方式
被谁调用、在什么场景下调用

## 约束
- 行为限制
- 输出规范
```

## 基本原则
- **独立**：每个 Agent 不依赖其他 Agent 的内部状态
- **声明式接口**：输入/输出清晰，orchestrator 据此编排
- **沙箱意识**：文件读写局限在 `research-assistant/` 内
- **技能复用**：优先调用父项目 `.claude/skills/` 中的专业技能（只读不写）
