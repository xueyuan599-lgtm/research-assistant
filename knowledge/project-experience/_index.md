# Project Experience — 项目经验沉淀

> 每次项目/竞赛结束后，复盘关键决策、踩坑记录、可复用模式。
> 沉淀在这里的经验是 AI 下次做类似项目时的重要参考。

## 条目格式

每条经验一个文件，命名格式：`<project-name>-<date>.md`

```markdown
# [项目名称]

- **时间**: 2026-07
- **方法**: [核心方法]
- **状态**: 已完成 / 进行中 / 搁置

## 目标
原本想做什么

## 实际方案
最终怎么做的，和原方案有哪些不同

## 踩坑记录
- 问题 + 原因 + 怎么解决的

## 可复用经验
下次做类似项目可以直接用的东西

## 代码/资源位置
相关文件路径
```

## 当前条目

（按时间倒序 — 暂无条目。完成项目后记得在此追加。）

---

> 💡 **提示**：经验不用写太长，每条 200-500 字即可。关键是 **踩坑** 和 **可复用** 两点。
