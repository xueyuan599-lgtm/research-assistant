# 数学建模优化问题：从求解到交付的完整方法论

> 基于 CUMCM 2024 C 题（农作物种植策略优化）实战总结。
> 涵盖：求解 → 可视化 → 验证 → LaTeX 论文 → 交付的全流程。

---

## 一、核心认知

### 1.1 "解出来"只是开始

大多数数模队伍止步于"求解器跑出结果"。但顶刊/竞赛高分的关键在后面的四个步骤：

```
求解器输出 → 可视化（让人看懂）→ 验证（证明解是好的）→ 论文（讲好故事）
    ↑_______________ 占 20% 工作量 _______________↑  占 80% 工作量
```

### 1.2 优化问题的三个层次

| 层次 | 问题 | 检查方式 |
|------|------|---------|
| **求解是否正确** | 求解器是否找到了全局最优？ | 状态码、对偶间隙 |
| **模型是否正确** | 约束是否完整描述现实？ | 可行性验证、经济合理性 |
| **解是否有用** | 能否指导决策？ | 影子价格、敏感性分析、基线对比 |

三个层次独立验证，缺一不可。

## 二、求解阶段要点

### 2.1 工具选择

| 场景 | 推荐工具 | 理由 |
|------|---------|------|
| 线性规划 (LP) | PuLP + CBC | 开源、Python 原生、速度快 |
| 整数规划 (MILP) | Gurobi / SCIP | CBC 在 MILP 上慢 |
| 非线性 | scipy.optimize / IPOPT | 取决于非凸程度 |
| 大规模 | CPLEX / Gurobi | 商业求解器性能优势明显 |

### 2.2 模型构建检查清单

- [ ] 决策变量定义清晰（含索引和上下界）
- [ ] 目标函数线性可分离
- [ ] 每个约束都有对应的现实含义注释
- [ ] 特殊约束（如轮作、时序）被正确编码
- [ ] 参数从数据正确映射（亩产、成本、价格）
- [ ] 随机种子固定（如涉及随机采样）

### 2.3 常见建模错误

| 错误 | 症状 | 修复 |
|------|------|------|
| 约束过紧 | 无可行解 (INFEASIBLE) | 放松约束，检查数据 |
| 约束过松 | 解"太好"不合常理 | 补遗漏的约束 |
| 单位不一致 | 利润数值量级异常 | 统一单位（元 vs 万元 vs 斤） |
| 索引错位 | 作物种到了不该种的地块 | 打印检查兼容性矩阵 |
| 边界条件遗漏 | 2023年历史数据未纳入轮作计算 | 补窗口边界 |

## 三、可视化方法论

### 3.1 数学建模论文需要哪些图

| 图类型 | 适用场景 | 这张图回答什么问题 |
|--------|---------|-------------------|
| **折线图** | 多方案逐年对比 | "哪种方案更好？趋势如何？" |
| **堆叠柱状图** | 收益/成本构成 | "利润从哪来？成本花在哪？" |
| **分组柱状图** | 情景对比 | "方案 A vs B 差距多大？" |
| **横向柱状图** | Top N 排名 | "哪些作物最重要？" |
| **饼图/环形图** | 结构占比 | "土地/资源如何分配？" |
| **热力图/矩阵** | 轮作/时序模式 | "时间和空间上的模式是什么？" |

### 3.2 图表质量标准

```
每张图必须满足：
1. 标题以"图N"开头，自解释（不看正文也能懂）
2. 坐标轴有单位和中文标签
3. 关键数值直接标注在图上（不是放图例里）
4. 配色色盲友好（蓝/红 > 绿/红）
5. 分辨率 ≥ 300 DPI
6. 两张图对比时，坐标轴范围一致

严禁事项（一票否决）：
7. ✗ 禁止在图中嵌入表格 —— 表格用 LaTeX table 环境，放入正文
   理由：图是图形层，表是文本层。混在一起导致：
   - 表格不能被 LaTeX 格式化（字体、行距、交叉引用）
   - 图片缩放时表格文字模糊
   - 审稿人无法复制表格数据
   - 违反学术出版规范
8. ✗ 禁止用 matplotlib table() 或 plt.text() 拼凑数据表
   替代方案：在图的 caption 中交叉引用正文中的表格（\ref{tab:xxx}）
```

### 3.3 本案例的 6 张图设计逻辑

```
图1 逐年利润折线      ← 回答"哪种方案赚得多" （总体趋势）
图2 收益成本堆叠      ← 回答"钱从哪来"       （结构分解）
图3 情景对比柱状+表   ← 回答"差异有多大"     （汇总对比）
图4 作物面积Top15     ← 回答"种了什么"       （决策内容）
图5 土地利用饼图      ← 回答"地怎么用"       （资源分配）
图6 轮作周期柱状      ← 回答"约束是否满足"   （约束验证）
```

覆盖了：**趋势、结构、对比、内容、资源、约束** 六个叙事维度。

### 3.4 避免的常见错误

- ❌ 英文标签（数模竞赛要求中文）
- ❌ 默认配色（matplotlib tab10 太丑）
- ❌ 图例盖住数据
- ❌ 坐标轴从非零开始但不标注（误导）
- ❌ 3D 图（除非数据真的是三维的）

## 四、六维验证框架

### 4.1 框架总览

```
Tier 1: 最优性证明 ← 数学确凿性
Tier 2: 可行性验证 ← 约束完整性
Tier 3: 经济合理性 ← 领域直觉
Tier 4: 敏感性分析 ← 数值稳定性
Tier 5: 基线对比   ← 改进幅度
Tier 6: 影子价格   ← 决策指导
```

### 4.2 各维度详解

#### Tier 1: 最优性证明

**核心问题**：求解器找到的解是不是全局最优？

- LP：单纯形法数学保证 → 只要返回 OPTIMAL 就是全局最优
- MILP：需要检查 optimality gap（Gurobi 的 MIPGap）
- 非线性/非凸：无法保证全局最优 → 需要多起点 + 记录

**输出**：
```
求解器状态: Optimal
目标函数值: 27,079,022 元
变量总数:   7,938
约束总数:   1,187
基变量数:   1,073（非零决策变量）
非基变量数: 6,865（在边界上=0）
```

#### Tier 2: 可行性验证

**核心问题**：解是否真的满足所有约束？

**方法**：从输出文件读取解，逐条检查每个约束类型，报告违规。

**必须检查的约束类型**：
1. 面积约束（sum ≤ capacity）
2. 兼容性约束（crop × land × season）
3. 时序约束（second ≤ first）
4. 数据存在性（有 yield 数据才有意义）
5. 轮作约束（3年窗口 ≥ 阈值）
6. 产销约束（sold ≤ min(prod, expected_sales)）

**常见陷阱**：Excel 输出格式可能与模型内部不一致（如本例中水稻在 Excel 中被标为"第一季"但模型中是"单季"）。

#### Tier 3: 经济合理性

**核心问题**：解是否符合领域直觉？

**检查项**：
- 高利润作物是否被优先种植？
- 低利润作物（如水稻）是否被替换？
- 亩均利润排序是否合理？（大棚 > 水浇地 > 旱地）
- 是否有明显矛盾？（如大米棚比蔬菜棚赚得多）

**本例验证结果**：
```
亩均利润梯度（情景一）:
  智慧大棚: 32,935 元/亩  ← 最高
  普通大棚: 28,372 元/亩
  水浇地:   15,110 元/亩
  平旱地:    2,700 元/亩  ← 最低
  ✓ 梯度合理：设施水平与利润正相关
```

#### Tier 4: 敏感性分析

**核心问题**：参数变化时，解是否稳定？

**两个层次**：
1. **定性**：识别哪个参数最关键（基于影子价格/经济学直觉）
2. **定量**：扰动参数 ±20%，重求解，观察目标函数变化

**本例发现**：
- 预期销售量是情景一的**紧约束**（瓶颈），最敏感
- 情景二对预期销售量不敏感（超产可半价卖出）
- 土地面积约束对大棚最紧（影子价格最高）

**判断标准**：
- 参数 ±10% → 利润变化 < 15%：解稳定 ✓
- 参数 ±10% → 利润变化 > 30%：解脆弱，需讨论 ⚠
- 参数 ±10% → 最优种植结构完全改变：模型过于敏感，需加入鲁棒性 ✗

#### Tier 5: 基线对比

**核心问题**：最优解比简单策略好多少？

**至少构造 3 个基线**：

| 基线 | 构造方法 | 检验什么 |
|------|---------|---------|
| **历史延续** | 沿用上一年方案 | 优化模型是否比"维持现状"更好 |
| **贪心策略** | 全部种利润最高的作物 | 约束条件是否真的改变了决策 |
| **均匀分配** | 每种作物平均分配土地 | 集中 vs 分散种植的价值 |

**判断标准**：
- 最优解 > 所有基线 → 优化有价值 ✓
- 最优解 ≈ 某基线 → 该基线已接近最优 OR 约束过紧限制了优化空间
- 最优解 < 某基线 → 基线计算有误 OR 优化模型有 bug（需排查！）

#### Tier 6: 影子价格分析

**核心问题**：如果要改进，该从哪入手？

**影子价格 = 约束右端项每增加 1 单位，目标函数的边际改善。**

| 约束类型 | 影子价格含义 | 决策指导 |
|---------|-------------|---------|
| 预期销售量 | 多卖 1 斤能多赚 X 元 | 拓展销售渠道 |
| 地块面积 | 多 1 亩地能多赚 X 元 | 开垦/租用土地 |
| 轮作约束 | 放松约束能省 X 元 | 评估轮作政策的真实成本 |
| 水浇地双季 | 多 1 亩第一季可支撑 Y 亩第二季 | 优化种植结构 |

**管理启示的写法**（论文中）：
> 影子价格分析表明，情景一的核心瓶颈是预期销售量约束。
> 以黄瓜为例，其影子价格为 4.2 元/斤，意味着每扩大 1 斤黄瓜的销售配额，
> 七年总利润可增加 4.2 元。建议优先拓展黄瓜、食用菌等高影子价格作物的销售渠道。

## 五、LaTeX 论文集成

### 5.1 图表插入规范

```latex
\begin{figure}[H]
    \centering
    \includegraphics[width=0.88\textwidth]{fig1_yearly_profit.png}
    \caption{2024—2030年逐年利润对比（情景一 vs 情景二）}
    \label{fig:yearly_profit}
\end{figure}

% 正文中引用
如图~\ref{fig:yearly_profit}所示，情景一的利润曲线近乎水平...
```

### 5.2 论文结构模板

```
封面 + 目录
一、问题重述     ← 用自己的话复述，不要照抄原题
二、问题分析     ← 识别优化类型、约束特征
三、模型假设     ← 编号列表，每条假设有理由
四、符号说明     ← 三线表：符号/含义/备注
五、模型建立与求解
    5.1 决策变量与目标函数  ← 公式 + 解释
    5.2 约束条件            ← 逐条公式 + 对应现实含义
    5.3 模型求解            ← 求解器 + 规模 + 时间
    5.4 求解结果与分析
        5.4.1 情景一 → 图1+图2+图4+图5
        5.4.2 情景二 → 图3
        5.4.3 轮作分析 → 图6
参考文献       ← ≥ 5篇
```

### 5.3 编译注意事项

- 中文论文必须用 **xelatex**（不是 pdflatex）
- 使用 `ctexart` 文档类
- 字体：正文宋体，标题黑体
- 图片格式：PNG（300 DPI）
- 交叉引用需编译两次

## 六、常见陷阱与解决方案

### 6.1 求解阶段

| 陷阱 | 发现方式 | 解决方案 |
|------|---------|---------|
| 产量-销量约束编码错误 | 利润值与预期严重不符 | 分别验证 scenario 1/2 的 objective |
| 轮作窗口边界遗漏 | 2023年数据未纳入计算 | 显式处理包含 2023 的窗口 |
| 统计数据缺失导致变量未创建 | 某些 crop-land-season 组合无解 | 打印 stat_lookup 的 key 覆盖率 |

### 6.2 输出阶段

| 陷阱 | 发现方式 | 解决方案 |
|------|---------|---------|
| Excel 显示季节与模型不一致 | 可行性验证（Tier 2）失败 | 添加后处理映射（如水稻 '第一季'→'单季'） |
| summary.json 计算与 LP objective 不一致 | 对比两者数值 | 统一 revenue 计算逻辑 |
| 图表中文乱码 | 预览时方块 | 设置 matplotlib 中文字体 |

### 6.3 验证阶段

| 陷阱 | 发现方式 | 解决方案 |
|------|---------|---------|
| 基线策略违反约束 | 基线利润"优于"最优解 | 检查基线是否 feasible |
| 影子价格为零但约束看似紧 | 检查对偶变量 | 可能约束实际不紧（slack > 0） |
| 可行性通过但约束公式写错 | 经济合理性检查（Tier 3） | 用极端 case 测试约束逻辑 |


> 四个环节都做到，才是完整的优化问题解答。
